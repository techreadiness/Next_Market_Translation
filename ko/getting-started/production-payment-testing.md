# Prod 실결제 테스트

{% hint style="warning" %}
&#x20;Prod 환경의 실결제 테스트에서는 Fiat 결제만 가능합니다.

실결제 테스트는 실제 결제가 발생하므로, 사전 협의 및 운영 유의사항을 반드시 준수해 주시기 바랍니다.
{% endhint %}

### 실결제 테스트 진행 가이드

Prod 환경에서 진행되는 실결제 테스트 절차를 설명합니다.

* 실결제 테스트에 대한 <mark style="color:$danger;">**별도의 테스트 비용 지원은 제공되지 않습니다**</mark>.&#x20;
*   테스트는 아래 순서로 진행됩니다.

    > **테스트 계획 사전 공유 → 넥스트 마켓 검토 및 협의 → 테스트 진행 → 결과 공유 → (필요 시) 결제 취소 처리**

### 1. 테스트 계획 사전 공유 (필수)

아래 항목을 정리하여 **테스트 희망일** **최소 3영업일 전 넥스트 마켓 측에 사전 공유**해야 합니다.

| 전달 항목        | 설명                                                                                                                                                                                                                                                                                             |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 테스트 범위       | <ul><li>대상 스토어</li><li><p>상품 구분</p><ul><li>B2C/C2C 구분</li><li>상품명 or sale_id</li></ul></li></ul>                                                                                                                                                                                               |
| 테스트 결제 수단 정보 | <ul><li><p>테스트 시 사용할 결제 수단 정보 입력</p><ul><li>카드 정보 </li><li>법인 카드/개인 카드</li><li>카드 명</li></ul></li></ul><p>⚠️ 결제 수단 별 결제 취소 가능 여부는 상이하여, <a href="production-payment-testing.md#undefined-3">결제 취소 정책</a> 및 <a href="production-payment-testing.md#undefined-1">취소 가능한 결제 수단</a> 확인 부탁드립니다.</p> |
| 테스트 결제 상세 정보 | <ul><li>결제 희망 건수 </li><li>결제 취소 희망 여부</li></ul>                                                                                                                                                                                                                                                |
| 테스트 일정 정보    | 테스트 예정 일시 (날짜 및 시간)                                                                                                                                                                                                                                                                            |
| 테스트 계정 정보    | 테스트 계정 GUID                                                                                                                                                                                                                                                                                    |

#### 결제 유의사항&#x20;

**\[ 결제 국가 및 통화 설정 ]** \
결제 화면 내 **청구지(Billing Address) 정보 설정**을 통해 결제 국가 및 통화를 선택하실 수 있습니다. 테스트하고자 하는 국가를 먼저 설정하신 후 결제를 진행해 주세요.

**\[ 국가별 결제 제한 안내 ]**\
일부 국가 및 결제 수단의 경우, **한국에서 발급된 신용카드로 결제가 제한**될 수 있습니다. 결제 테스트 또는 실결제 진행 전 아래 내용을 반드시 확인해 주세요.

* **대만**
  * 결제 수단: **MyCard**
  * 한국에서 발급된 신용카드로는 결제가 지원되지 않습니다.
* **미국**
  * 결제 수단: **Payletter / Coda**
  * 한국에서 발급된 신용카드로는 결제가 지원되지 않습니다.

\
해외 결제 테스트가 필요한 경우, 해당 국가에서 발급된 결제 수단 또는 지원 가능한 대체 결제 수단을 사전에 준비해 주시기 바랍니다.

#### 결제 취소가 가능한 결제 수단

| 지역 (청구지) | 상품의 통화                      | 결제 수단                                                                        |
| -------- | --------------------------- | ---------------------------------------------------------------------------- |
| 한국       | KRW                         | 신용카드                                                                         |
| 한국       | KRW                         | 간편결제                                                                         |
| 대만       | TWD                         | <p>MyCard PG에서 제공하는 신용카드/휴대폰결제/전자결제수단<br>⚠️ MyCard point 는 제공 및 결제 취소 불가</p> |
| 글로벌      | USD / JPY / TWD / THB / PHP | <p>Paypal (신용카드)<br>⚠️ Paypal 결제 진행 시 Paypal 회원만 결제 가능</p>                   |

#### 결제 취소 정책&#x20;

* 결제 수단별로 결제 취소 가능 여부는 상이합니다.
* 취소 가능한 수단에 한해, 사전 협의된 절차에 따라 결제 취소 처리가 가능합니다.&#x20;
* 결제 취소를 희망하지 않는 경우, **취소 불가 결제 수단으로도 테스트 진행이 가능합니다.**
* 결제 취소 시 <mark style="color:$danger;">**환불 비용이 발생**</mark>할 수 있습니다. 해당 환불 비용은 개발사와 LINE NEXT 가 공동 부담합니다.&#x20;

### 2. 넥스트 마켓 검토 및 협의

* 공유된 테스트 계획을 검토합니다.
* 결제 수단 및 취소 가능 여부를 포함하여 **실결제 테스트 진행 가능 여부를 확정**한 뒤 개발사에 공유합니다.

### 3. 테스트 진행

#### 넥스트 마켓 준비사항

① 테스트 대상 상품 설정

* 상품 노출은 설정된 **상품 판매 시작 시간**을 기준으로 노출됩니다.
* 테스트를 위해 상품 판매 시작 시간을 **테스트 시간으로 임시 수정**해야 합니다.

② 점검 모드 설정 (권장 사항)

* 실결제 테스트는 **Prod 환경에서만 진행**됩니다.
* 일반 유저 접근을 제한하고 개발사만 테스트를 진행할 수 있도록 **점검 모드 전환 후 진행을 권장**합니다.&#x20;
* 점검 모드 상태에서도 테스트가 가능하도록 **점검 우회 링크를 개발사에 제공합니다.**

{% hint style="warning" %}
**점검 모드 미 설정 시**&#x20;

서비스 점검 모드로 설정하지 않은 상태에서 실결제 테스트를 진행할 경우, 일반 사용자도 결제 페이지에 접근 및 결제가 가능합니다. 실제 판매 기간 중 테스트가 필요하다면, 유저 결제가 발생할 수 있으니 사전에 결제 테스트 일정 협의 시 참고 부탁드립니다.
{% endhint %}

#### 개발사 테스트 진행

* 사전 협의된 **결제 수단 및 결제 건수** 기준으로 실결제 테스트를 진행합니다.

### 4. 테스트 결과 공유

* 테스트 완료 후 아래 정보를 넥스트 마켓에 공유해야 합니다.
  * 테스트 계정의 GUID
  * 사용 결제 수단
  * 결제 정보 (order\_id, 결제 금액)

### 5. 결제 취소 처리

* 결제 취소가 가능한 수단에 한해 결제 취소를 희망하는 경우, 테스트 결과에 포함된 주문 건 전체에 대해 취소 처리를 진행합니다.
* 취소 요청은 넥스트 마켓 개발팀에서 처리합니다.
